# Contributing to AI Lecture Assistant

Thank you for your interest in contributing! Here's how you can help.

## 🚀 Getting Started

1. **Fork** the repository
2. **Clone** your fork:
   ```bash
   git clone https://github.com/yourusername/ai-lecture-assistant.git
   ```
3. Create a **branch** for your feature:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. Make your changes
5. **Test** your changes locally
6. **Commit** with a clear message:
   ```bash
   git commit -m "feat: add support for audio trimming"
   ```
7. **Push** and create a Pull Request

## 📝 Commit Convention

Use conventional commit messages:

- `feat:` — New feature
- `fix:` — Bug fix
- `docs:` — Documentation changes
- `style:` — CSS/formatting changes
- `refactor:` — Code restructuring
- `test:` — Adding tests
- `chore:` — Build/tooling changes

## 🐛 Reporting Bugs

Open an issue with:
1. **Title** — Brief description
2. **Steps to reproduce** — What you did
3. **Expected behavior** — What should have happened
4. **Actual behavior** — What actually happened
5. **Screenshots** — If applicable
6. **Environment** — OS, Python version, browser

## 💡 Feature Requests

Open an issue with the `enhancement` label describing:
1. The problem you're trying to solve
2. Your proposed solution
3. Any alternatives you've considered

## 📋 Code Style

- **Python**: Follow PEP 8
- **HTML/CSS/JS**: Use consistent indentation (4 spaces)
- **Comments**: Write meaningful comments for complex logic
- **Functions**: Keep functions focused and under 50 lines when possible

## 🔒 Security

- Never commit API keys or secrets
- Always use environment variables for sensitive configuration
- Report security vulnerabilities privately via email

## ✅ PR Checklist

Before submitting a PR, ensure:
- [ ] Code works locally
- [ ] No API keys or secrets in the code
- [ ] Existing features still work
- [ ] Added comments for complex changes
- [ ] Updated documentation if needed
